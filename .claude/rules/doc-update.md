# ドキュメント更新チェック

site/ や pipeline/ のコードを変更した場合、以下のドキュメントに更新が必要かを確認すること:

1. **CLAUDE.md** — サイト構成、技術スタック、データソースの記述が実態と一致しているか
2. **.claude/rules/site.md** — ページ一覧、アーキテクチャ記述
3. **.claude/rules/pipeline.md** — スクレイパーパターン、タグ生成の記述
4. **.claude/rules/data.md** — データ構成、スキーマ記述

## 判断基準

以下に該当する変更があった場合、ドキュメント更新が必要:

- ページの追加・削除・パス変更
- 技術スタック（依存パッケージ）の追加・削除
- データスキーマの変更
- CI ワークフローの変更
- ホスティング・デプロイ設定の変更

軽微な UI 調整やバグ修正ではドキュメント更新は不要。
