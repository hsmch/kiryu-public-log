# PR レビューコメントの修正

現在のブランチに紐づく PR のレビューコメントを取得し、指摘内容に基づいて修正を行う。

## 手順

1. 現在のブランチ名を取得する
2. `gh pr view --json number,url` で PR 番号を取得する（PR が見つからなければ中止）
3. `gh api repos/{owner}/{repo}/pulls/{number}/comments` でレビューコメントを取得する
4. 各コメントについて:
   - 対象ファイル (`path`) と行番号 (`line`) を特定する
   - コメント本文 (`body`) から指摘内容を理解する
   - `suggestion` コードブロックがあればその内容を優先的に採用する
   - 対象ファイルを読み、指摘に基づいて修正する
5. 修正内容をまとめてコミットする（コミットメッセージに修正したレビューの概要を含める）
6. push する

## 注意事項

- bot（Copilot 等）のコメントも人間のコメントも同様に扱う
- 指摘が事実と異なる場合は修正をスキップし、その旨を報告する
- suggestion ブロックの内容はそのまま適用するのではなく、文脈を確認してから適用する
